# 桌面控制台 Windows 打包教程

![应用图标](./cn/app-icon.png)

## 1. 文档目的

本文档用于指导开发者在 Windows 环境下将 `Desktop Console / 桌面控制台` 打包为可分发的桌面应用。
当前项目基于以下技术栈：

- Electron Forge
- Vite
- React 19
- TypeScript
- better-sqlite3

打包完成后，默认产物位于：

```text
out/Desktop Console-win32-x64/Desktop Console.exe
```

## 2. 应用简介

桌面控制台是一个本地优先的桌面入口管理工具，用来统一管理：

- 常用网站
- 本地应用程序
- Windows 快捷方式（`.lnk`）
- 分类导航
- 快速呼出与快捷键访问

它的目标是让用户用一个统一界面来组织和启动日常工作入口。

## 3. 打包前准备

### 3.1 系统要求

建议环境：

- Windows 10 / Windows 11
- Node.js 22.x
- npm 10.x
- Visual Studio 2022 Build Tools

### 3.2 必装组件

由于项目依赖 `better-sqlite3`，Windows 上需要可用的 C++ 构建工具链。
请安装：

- `Visual Studio 2022 Build Tools`
- 工作负载：`Desktop development with C++`

建议同时勾选：

- `MSVC v143 - VS 2022 C++ x64/x86 build tools`
- `Windows 10/11 SDK`
- `C++ core features`

### 3.3 依赖安装

在项目根目录执行：

```powershell
npm.cmd install
```

## 4. 开发态验证

建议在打包前先验证开发环境可正常运行：

```powershell
npm.cmd run typecheck
npm.cmd run lint
npm.cmd run start
```

说明：

- `typecheck` 用于检查 TypeScript 类型
- `lint` 用于检查代码规范
- `start` 用于启动 Electron 开发环境

如果开发态无法运行，请先修复再打包。

## 5. 正式打包

### 5.1 执行打包命令

```powershell
npm.cmd run package
```

打包成功后，产物默认输出到：

```text
out/Desktop Console-win32-x64/
```

主程序文件为：

```text
out/Desktop Console-win32-x64/Desktop Console.exe
```

### 5.2 可选：生成安装包

如果你后续需要安装包而不是单独的 exe 目录，可以继续执行：

```powershell
npm.cmd run make
```

说明：

- `package` 更适合本地自测或直接复制目录分发
- `make` 更适合生成安装器或可发布制品

## 6. 打包后自测建议

建议至少完成以下冒烟测试：

1. 双击 `Desktop Console.exe`，确认主窗口可正常打开
2. 检查托盘菜单是否显示正常中文
3. 检查全局快捷键是否可正常注册
4. 新建一个网站卡片，确认可正常打开
5. 拖入一个 `.exe` 或 `.lnk`，确认可正确生成应用卡片
6. 检查中文路径应用是否显示正常

## 7. 常见问题

### 7.1 better-sqlite3 相关报错

如果出现原生模块编译或加载错误，优先检查：

- 是否安装了 `Visual Studio 2022 Build Tools`
- 是否安装了 C++ 工作负载
- 是否重新执行过 `npm.cmd install`

### 7.2 打包后白屏

如果出现窗口能打开但界面为空，请确认：

- 打包时没有混入开发态产物
- 使用的是最新重新打包后的 `out/Desktop Console-win32-x64/Desktop Console.exe`
- 没有同时残留多个旧版本进程

### 7.3 中文路径乱码

如果旧数据中存在历史乱码路径，请使用最新打包版本重新启动一次，让应用自动回补元数据。

## 8. 推荐发布流程

推荐顺序：

1. `npm.cmd install`
2. `npm.cmd run typecheck`
3. `npm.cmd run lint`
4. `npm.cmd run package`
5. 手动运行打包后的 exe 做自测
6. 确认无误后再上传到 GitHub Release 或其他分发渠道

## 9. 目录说明

```text
assets/                       应用图标资源
src/main/                     主进程代码
src/renderer/                 渲染进程代码
docs/                         项目文档
out/Desktop Console-win32-x64 打包输出目录
```
